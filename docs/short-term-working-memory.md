# Short-Term Working Memory

更新时间：2026-06-04

## 1. 当前阶段

当前短期记忆主线已覆盖：

- Phase 1：raw event 记录与基础只读接口
- Phase 2：规则型 working item 提炼与分类 summary
- Phase 3：显式 compaction 与 archive
- Phase 4：archive detail、run history、maintenance worker、runtime observability
- Phase 5：promotion candidate 生成、存储与只读展示
- Phase 5.1：promotion candidate review readiness
- Phase 6：promotion candidate review / accept
- Phase 7：promotion feedback signal / draft learning case foundation
- Phase 8：stable review readiness
- Phase 9：stable review accept / reject
- Phase 10：stable memory integration / provenance verification
- Learning Loop Phase 1：promotion review 自动生成 learning records / cases

本阶段仍不改 retrieval 语义，不改 scoring / PackingPolicy，不接 vector，不引入 layered retrieval，不做 LLM promotion judge，不做自动 StableMemory promotion。Stable Review `generate` 仍只生成人工 review candidate；只有显式 accept endpoint 会在重新 validation 通过后写入稳定目标层。

## 7. Review / Accept 规则

accept 行为：

- `Candidate` / `NeedsReview` 可视为 pending review 状态；`Pending` 是 `Candidate` 的兼容别名
- accept 后状态更新为 `Accepted`
- 记录 `reviewer` / `reason` / `reviewedAt`
- `CandidateMemory` 写入 `ContextMemoryLayer.Structured + ContextMemoryStatus.Candidate`
- `DecisionRecord` 写入候选记忆，`Type=decision`
- `KnownIssue / OpenIssue` 写入候选记忆
- `ConstraintCandidate` 写入 `ContextConstraint`，`Status=Candidate`
- accept result 返回 `CreatedTargetItemId`
- 不写入 StableMemory
- 已 accepted / rejected / expired 的 candidate 不允许再次 accept

reject 行为：

- `Candidate` / `NeedsReview` 可拒绝
- reject 后状态更新为 `Rejected`
- 记录 `reviewer` / `reason` / `reviewedAt`
- 不删除 candidate

evidence chain：

- target metadata 包含 `sourceCandidateId`
- target metadata 包含 `sourceWorkingItemId`
- target metadata 包含 `evidenceRefs`
- 如 relation store 可用，写入 `promoted_from`、`derived_from`、`evidence_for`

review history：

- accept / reject / expire 都记录 `PromotionCandidateReviewRecord`
- record 包含 `reviewer` / `reason` / `reviewedAt`
- candidate 不会被删除，只更新 status 和 review metadata

learning feedback：

- accept 生成 `PromotionFeedbackSignal(Action=Accepted)`
- reject 生成 `PromotionFeedbackSignal(Action=Rejected)`
- accept 生成 positive draft `ContextLearningCase`
- reject 生成 negative draft `ContextLearningCase`
- accept 自动生成 `PromotionAccepted / Positive` learning record
- reject 自动生成 `PromotionRejected / Negative / PromotionFalsePositive` learning record 和 `PromotionFalsePositive` case
- expire 自动生成 `PromotionExpired / Stale / StaleCandidate` learning record
- learning 只记录反馈，不训练模型，不自动调参，不改变 retrieval

target consistency validation：

- accepted candidate 的 `CreatedTargetItemId` 必须存在
- target item 必须带 source candidate ref
- target status 必须是 `Candidate`
- target 不得直接写入 StableMemory

## 8. Stable Review Readiness

`StableReviewCandidate` 用于表示 accepted promotion candidate 进入 Stable review 前的只读候选项。

字段包括：

- `StableReviewCandidateId`
- `WorkspaceId`
- `CollectionId`
- `SessionId`
- `SourceCandidateId`
- `SourceTargetItemId`
- `SourceLearningCaseId`
- `Kind`
- `Title`
- `Summary`
- `SuggestedStableTarget`
- `Reason`
- `Confidence`
- `Importance`
- `EvidenceRefs`
- `RiskFlags`
- `ValidationStatus`
- `CreatedAt`
- `Status`
- `Metadata`

生成规则：

- 只扫描 `PromotionCandidateStatus.Accepted`
- `CandidateMemory -> StableMemory`
- `ConstraintCandidate -> StableConstraint`
- `DecisionRecord -> DecisionRecord`
- `OpenIssue / KnownIssue` 默认不直接生成 stable review，除非 metadata 显式标记 `stableReviewAllowed` / `allowStableReview` / `generateStableReview` / `stableReview`

validation：

- `evidenceRefs` 不能为空，否则 `ValidationStatus=NeedsMoreEvidence`
- source candidate 必须存在
- source target item 必须存在
- target item status 必须是 `Candidate`
- 不允许 duplicate stable
- 不允许 scope mismatch
- 不允许 lifecycle conflict

当前 status：

- `Candidate`
- `NeedsMoreEvidence`
- `Blocked`
- `Accepted`
- `Rejected`

当前 validation status：

- `ReadyForReview`
- `NeedsMoreEvidence`
- `SourceCandidateMissing`
- `SourceTargetMissing`
- `TargetNotCandidate`
- `DuplicateStableCandidate`
- `ScopeMismatch`
- `LifecycleConflict`

FileSystem 路径：

- `memory/stable-review-candidates.jsonl`
- `memory/stable-review-candidate-reviews.jsonl`

## 9. Stable Review Accept / Reject

Stable Review accept / reject 是人工 opt-in 操作，不由 generator 自动触发。

accept 前会重新执行 validation：

- `evidenceRefs` 不为空
- source promotion candidate 存在且已 accepted
- source target item 存在
- target item status 为 `Candidate`
- validation status 为 `ReadyForReview`
- no duplicate stable
- no scope mismatch
- no lifecycle conflict

accept 后写入稳定目标层：

- `StableMemory`：写入 `ContextMemoryLayer.Stable + ContextMemoryStatus.Stable`
- `DecisionRecord`：写入 stable memory，`Type=decision`
- `StableConstraint`：写入 `ContextConstraint`，`Status=Stable`
- 第一版不写 `GlobalMemory`

stable item 必须保留：

- `sourceStableReviewCandidateId`
- `sourcePromotionCandidateId`
- `sourceLearningCaseId`
- `sourceWorkingItemId`
- `sourceFeedbackId`
- `sourceTargetItemId`
- `evidenceRefs`
- `reviewer`
- `reviewReason`
- `policyVersion`
- `createdFrom=stable_review_accept`

reject 行为：

- 只把 Stable Review candidate 状态更新为 `Rejected`
- 记录 `StableReviewRecord`
- 不删除 candidate
- 不写 stable target

review history：

- accept / reject 均记录 `StableReviewRecord`
- record 包含 action、from/to status、reviewer、reason、validation status、risk flags、stable target、source refs、evidence refs、reviewedAt

## 10. Stable Provenance

Phase 10 新增统一来源链查询：

- `GET /api/provenance/{itemId}`
- 支持可选 `workspaceId` / `collectionId` 用于普通 memory / constraint id 定位
- `stable:mem:{StableReviewCandidateId}` / `stable:decision:{StableReviewCandidateId}` / `stable:constraint:{StableReviewCandidateId}` 可直接反推出 Stable Review candidate

返回内容包括：

- target memory / constraint item
- stable review candidate
- promotion candidate
- promotion feedback signal
- learning case
- source working item
- evidence refs
- stable review history
- promotion review history
- warnings / missing links
- stable diagnostics

只读 diagnostics 当前覆盖：

- duplicate stable warning
- possible conflict warning
- missing source link warning

该 endpoint 只做 provenance 聚合和只读诊断，不做自动修复、不训练模型、不改变 retrieval。
